Logistics-KE

Kenya's transport marketplace — connecting cargo owners with available transporters across the country. Post a job, get matched instantly, or invite competitive bids. All communication by SMS and WhatsApp. No app download required.

🚚
For transporters
Grow your haulage business without middlemen
  • Register your vehicle, capacity, and rate per kilometre — takes under a minute.
  • Instantly see cargo jobs posted near your base that match your vehicle and capacity.
  • Accept instant jobs at your listed rate, or submit a competitive bid on open-tender requests.
  • Pickup address, drop-off address, cargo weight, and landmark directions are all shown upfront — no surprises.
  • Receive an SMS and WhatsApp message the moment a new job matches your vehicle.
  • Your profile shows your availability to clients browsing the platform from any phone.
📦
For clients
Move your cargo reliably and at the right price
  • Post a cargo request in under 2 minutes — describe your pickup, drop-off, cargo type, and weight.
  • Pin exact locations on a live map and add landmark notes so transporters know precisely where to go.
  • Instant booking — the system automatically matches you with the nearest available transporter who fits your cargo. No waiting.
  • Request for quotes (RFQ) — post your job and let transporters bid on it. Review all bids, accept the best price, or counter-offer. A floor rate on every vehicle type prevents bids from going unsustainably low.
  • Once booked, confirm directly with the transporter via SMS or WhatsApp — tap a single button to open a pre-filled message on your phone.
  • Browse all registered transporters near you even before posting a job.

🚀 Deploy to the Web (GitHub Pages)

What this gives you

A permanent public URL — e.g. https://yourname.github.io/logistics-ke/ — that anyone in Kenya can open on any phone or laptop. No hosting fees, no subscription.

Step 1 — Create a GitHub account

  1. Go to github.com and click Sign up.
  2. Enter any email, choose a username (e.g. logistics-ke), and set a password.
  3. Verify your email address from the confirmation email GitHub sends you.

Step 2 — Create a new repository

  1. Click the + icon in the top-right of GitHub → New repository.
  2. Repository name: logistics-ke
  3. Set visibility to Public.
  4. Tick "Add a README file".
  5. Click Create repository.

Step 3 — Upload the app file

  1. Inside your new repository, click Add file → Upload files.
  2. Drag and drop logistics-ke.html into the upload area (the file you downloaded from this tool).
  3. Also upload logistics-ke-backend.gs.txt if you want to keep the backend script safe in the same repo (optional — it won't be served publicly).
  4. In the Commit changes box at the bottom, write Initial upload.
  5. Click Commit changes.

Step 4 — Rename the file so it loads as your homepage

  1. In your repository, click on logistics-ke.html.
  2. Click the pencil (✏️ Edit) icon on the right.
  3. At the top of the editor you'll see the filename. Change logistics-ke.html to index.html.
  4. Scroll down and click Commit changes.
💡 Naming it index.html means GitHub Pages will load it automatically when someone visits your URL — no need to type the filename.

Step 5 — Turn on GitHub Pages

  1. In your repository, click Settings (top menu, last item).
  2. In the left sidebar, click Pages.
  3. Under Source, select Deploy from a branch.
  4. Under Branch, select main and leave the folder as / (root).
  5. Click Save.
  6. Wait 60–90 seconds, then refresh the Pages settings page.
  7. You will see a green banner: "Your site is live at https://yourname.github.io/logistics-ke/"

Step 6 — Share your live link

Your URL looks like: https://YOUR-GITHUB-USERNAME.github.io/logistics-ke/

Share this link via WhatsApp, SMS, or a QR code. Anyone who opens it on any phone gets the full app — already connected to your Google Sheet.

⚠ The admin passphrase is inside the HTML file. Do not share it with anyone. Only you need to enter it. To change it, edit line 420 in the file (const ADMIN_SECRET=) before uploading to GitHub.

Step 7 — Updating the app in future

  1. Download the latest logistics-ke.html from this tool.
  2. In your GitHub repository, click on index.html.
  3. Click the pencil ✏️ icon → then click Edit.
  4. Select all the text (Ctrl+A / Cmd+A) and delete it.
  5. Open the new logistics-ke.html in a text editor, select all, copy, and paste into GitHub's editor.
  6. Click Commit changes. Your live site updates in ~30 seconds.
✓ All existing user data in your Google Sheet is untouched — updates only change the app, not the data.

🗂 Backend setup reminder (Google Sheet)

Your Google Sheet URL is already wired into this app. The backend script (logistics-ke-backend.gs.txt) must be deployed as described below if not already done.

  1. Open sheets.google.com → create a blank sheet named Logistics-KE Database.
  2. Click Extensions → Apps Script. Delete starter code, paste the full contents of logistics-ke-backend.gs.txt.
  3. Click Save (name project Logistics-KE API).
  4. In the function dropdown, select initialSetup → click ▶ Run. Authorize when prompted (it's your own script — safe to allow).
  5. Click Deploy → New deployment → Web app. Set Execute as: Me, Who has access: Anyone. Click Deploy.
  6. The Web app URL shown is already the one wired into this app — no further action needed.
